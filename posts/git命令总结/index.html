git命令总结

02 Apr 2017
@nzooherd

Java选修课上，老师建议采用git来管理自己的代码．其实很久以前就了解到这个linus大神编写的版本控制工具,也看过一些它的命令，现在抽个时间整理（抄袭）一下，大部分来自阮一峰大大．

首先记一下图中的6个命令．

Workspace: 工作区
Index: 缓存区
Reository: 本地仓库
Remote: 远程仓库

新建代码库

$ git init  //在当前目录新建一个git代码仓库
$ git init [project_name] //新建一个目录，将其初始化为Git代码库  
$ git clone ［url]  //下载一个项目和它的整个代码历史

配置

$ git config --list //显示当前的Git配置
$ git config -e [--global] //编辑配置文件

增加删除文件

$ git add [file] //添加指定文件到缓存区
$ git add [dir] //添加指定目录到缓存区
$ git add //添加当前目录所有文件到缓存区
$ git add -p //对于同一个文件的多处变化，可以实现分次提交(不是很懂)
$ git rm [file] //删除工作区文件，并将这次删除放入暂存区
$ git mv [file-orginal] [file-renamed] //修改文件名

代码提交

$ git commit -m [message] //提交暂存区到仓库区
$ git commit [file1] [file2] -m [messae] //提交暂存区的指定文件到仓库区
$ git commit -a //提交工作区自上次commit之后的变化，直接到仓库区
$ git commit -v　//显示所有的diff信息．

分支

$ git branch //列出所有本地分支
$ git branch -r //列出所有远程分支
$ git branch -a //列出所有本地分支和远程分支

查看信息

$ git status //显示所有变更的文件
$ git log  //现实当前分支的历史版本
$ git log --stat  //显示commit历史
$ git log -S [keyboard] //根据关键字搜索历史
$ git reflog //现实当前分支的最近几次提交

远程同步

$ git fetch [Remote] //下载远程仓库的所有变动
$ git remote -V //显示所有远程仓库
$ git remote show [remote] //显示某个远程仓库的信息
$ git remote add [shortname] [url] //增加一个新的远程仓库，并命名
$ git pull [remote] [branch] //取回远程仓库的变化，并与本地分支合并
$ git push [remote] [branch] //上传本地指定分支到远程仓库
$ git push [remote] --force  //强行推送当前分支到远程仓库，即使有冲突
$ git push [remote] --all //推送所有分支到远程仓库

撤销

$ git checkout [file] //恢复暂存区的指定文件到工作区  
$ git checkout [commit] [file] //回复某个commit的指定文件到暂存区  
$ git checkout //恢复暂存区的所有文件到工作区
$ git stash       //暂时将未提交的变化移除       
$ git stash pop   //再移入，接上一条命令

以上就是我觉得我可能用到的一些命令，强行从别人的网站复制粘贴了过来，根据二八定律,估计常用的只有小部分．
